> MBA Term Project · 팀 내부 공유용

---

## 1. 프로젝트 개요

### 서비스 정의
Korea Law MCP(국가법령정보센터 실시간 연동)를 기반으로, **법령 개정 모니터링 → 사내 규정 영향 분석 → 개정안 초안 작성 → 판례 기반 리스크 검토 → 결재용 사유서 생성**까지의 컴플라이언스 프로세스를 End-to-End 자동화하는 B2B SaaS.

### One-Liner
> "법령이 바뀌면, 우리 회사 규정이 스스로 바뀐다."

---

## 2. 문제 정의 (Problem)

### 실무 현장의 Pain Point
1. **모니터링 불가능성**: 연간 수백 건의 법령 개정 중 어떤 것이 자사 규정에 영향을 주는지 실무진이 모두 추적하는 것은 불가능
2. **문서 작업 과부하**: 규정 하나를 개정하려면 신구조문대비표, 개정 사유서, 주요 골자를 수작업으로 작성해야 함
3. **부서 간 Silo**: 인사·법무·보안·총무 규정이 파편화되어 서로 모순되는 조항 발생
4. **경영진 책임 확대**: 중대재해처벌법·개인정보보호법 등으로 규정 불일치가 곧 경영진 개인 책임으로 연결

### 기존 해결 방식의 한계
- 법무법인 자문: 비용 높고, 지속적 모니터링 불가
- 기존 법령 검색 서비스(로앤비 등): 검색만 될 뿐 사내 규정과 연결 안 됨
- 사내 법무팀: 인력 한계로 사후 대응에 그침

## 3. 솔루션 (Solution)

### 5대 핵심 기능

| # | 기능 | 핵심 아웃풋 | 우선순위 |
|---|------|-------------|----------|
| 1-A | 📡 법령 개정 디텍팅 (스캔 모드) | 영향받는 규정 리스트 + 규정별 개정 필요 사유 및 법적 근거 | **P0** |
| 1-B | 📝 개정안 자동 생성 + 판례 검증 (선택 모드) | 개정 사유 / 주요 골자 / 신구조문대비표 / 판례 교차검증 리포트 | **P0** |
| 1-C | 🔗 연쇄 영향 탐지 (도미노 체크) | 개정된 규정으로 인해 추가로 바꿔야 할 다른 규정 + 사유 | **P0.5** |
| 3 | 🎯 경영 방향성 기반 전사 규정 일괄 정렬 | 일괄 개정안 + 신구조문대비표 | P2 |
| 4 | ⚖️ 내부 규정 간 충돌 탐지 | 충돌 지점 + 중재안 | P1 |
| 5 | 🤖 내/외부 통합 컴플라이언스 챗봇 | 규정+판례 엮은 실시간 행동 가이드 | P1 |

### 핵심 파이프라인 (2단계 구조)

> 설계 원칙: AI가 알아서 다 바꾸는 게 아니라, **AI가 찾아주고 → 사람이 선택하고 → AI가 작성**하는 구조.

```
━━━ Step 1: 디텍팅 (스캔 모드) ━━━

[법령 개정 이벤트 감지]
        ↓ (Korea Law MCP 폴링)
[사내 규정 10개 전수 매칭]
        ↓ (임베딩 기반 의미 매칭)
[영향받는 규정 리스트 + 개정 필요 사유 알림]
  → "취업규칙 제12조: 근로기준법 제50조 개정으로 근로시간 기준 변경. 현행 규정이 구법 기준이므로 개정 필요"

        ↓ 🧑 사용자가 대상 규정 선택

━━━ Step 2: 개정안 생성 + 판례 검증 (선택 모드) ━━━

[선택된 규정 × 해당 법령 집중 분석]
        ↓ (LLM + RAG)
[대법원 판례 DB 교차검증]
        ↓
[결재용 패키지 출력]
  = 개정 사유 + 주요 골자 + 신구조문대비표 + 판례 검증 리포트

━━━ Step 3: 연쇄 영향 탐지 (도미노 체크) ※ P0.5 ━━━

[개정된 규정이 다른 사내 규정에 영향을 주는지 재스캔]
        ↓
[추가 영향 규정 리스트 + 사유 알림]
  → "취업규칙 개정으로 인해 복무규정 제4조, 징계규정 제7조도 검토 필요"
```

## 4. 타겟 고객 & 비즈니스 모델

### 1차 타겟 (규제 민감 산업)
- **제약**: 약사법, GMP, 임상시험 관련 규정 변화 빈번
- **금융**: 자본시장법, 개인정보보호법, 금소법
- **공공기관**: 청탁금지법, 공공감사법, 이해충돌방지법
- **대기업 법무/준법감시부서**: 중대재해처벌법 대응

### 과금 모델 (SaaS)
- **Starter**: 월 정액 (중소기업, 규정 100건 이하)
- **Enterprise**: 규정 건수 기반 + API 호출량
- **Add-on**: 판례 검증 / 챗봇 / 충돌 탐지 모듈별 과금

## 5. 차별화 포인트

### vs 기존 법령 검색 서비스
→ 검색이 아니라 **사내 규정과 자동 연결**

### vs 챗GPT 프롬프팅
→ **MCP 기반 실시간 공식 데이터** 연동 (hallucination 방지)

### vs 법무법인 자문
→ **24/7 상시 모니터링** + 비용 1/10 수준

## 6. MVP 범위 (발표 기준)

### 완성도 있게 시연할 것
- ✅ 기능 1-A (사내 규정 10개 스캔 → 영향 규정 디텍팅 + 사유/근거)
- ✅ 기능 1-B (선택한 규정 개정안 생성 + 판례 검증)

### Mockup/Wireframe만 제시할 것
- 기능 3, 4, 5

### 시연 시나리오 (2단계 흐름)
> 1. 사내 규정 10개 세팅 (취업규칙, 인사규정, 보안규정 등)
> 2. "최근 법령 개정 스캔" 실행
> 3. → "취업규칙 외 2건 영향받음" 알림 + 각 규정별 왜 바꿔야 하는지 사유·근거 조문 표시
> 4. 사용자가 "취업규칙" 선택 → "개정안 추천" 클릭
> 5. → 개정 사유 + 주요 골자 + 신구조문대비표 + 판례 검증 리포트 출력
