# MANIFESTO: 레귤메이트 (RegulMate)

> 실시간 법령-사내규정 동기화 AI 솔루션

---

## 🎯 One-Liner

Korea Law MCP로 국가법령정보센터와 실시간 연동하여, 법령 개정 모니터링부터 사내 규정 개정안 작성까지의 컴플라이언스 프로세스를 자동화하는 B2B SaaS.

---

## 💡 해결하는 문제 (Problem)

- 외부 법령은 수시로 개정되지만, 기업 실무진이 이를 매번 트래킹하는 것은 불가능
- 사내 규정 개정 작업(신구조문대비표, 개정 사유서 등)은 문서 작업량이 과도하게 큼
- 부서별로 파편화된 규정들이 서로 모순되는 Silo Effect 발생

---

## 🚀 핵심 가치 (Value Proposition)

### 2단계 파이프라인

**Step 1: 디텍팅 (스캔 모드)**
- 법령 개정 감지 → 사내 규정(10개~) 임베딩 매칭
- 아웃풋: 영향받는 규정 리스트 + **규정별 개정 필요 사유 및 법적 근거**
- 알림 예시:
  - "취업규칙 제12조 → 근로기준법 제50조 개정으로 근로시간 기준 변경. 현행 규정이 구법 기준이므로 개정 필요"
  - "개인정보처리규정 제8조 → 개인정보보호법 시행령 제30조 개정. 보유기간 조항 불일치"

**Step 2: 개정안 생성 + 판례 검증 (선택 모드)**
- 사용자가 영향받는 규정 중 대상을 선택 → "개정안 추천" 실행
- 해당 규정 × 해당 법령만 집중 분석 + 대법원 판례 DB 교차 검증
- 아웃풋: 개정 사유 + 주요 골자 + 신구조문대비표 + 판례 검증 리포트 (결재용 패키지)

> 핵심 설계 원칙: AI가 알아서 다 바꾸는 게 아니라, **AI가 찾아주고 사람이 선택하고 AI가 작성**하는 구조. 실무 신뢰도와 기술 현실성을 동시에 확보.

## ⚙️ 5대 핵심 기능

| # | 기능 | 핵심 아웃풋 |
|---|------|-------------|
| 1-A | 📡 법령 개정 디텍팅 (스캔 모드) | 영향받는 규정 리스트 + 규정별 개정 필요 사유 및 법적 근거 |
| 1-B | 📝 개정안 자동 생성 + 판례 검증 (선택 모드) | 개정 사유 / 주요 골자 / 신구조문대비표 / 판례 교차검증 리포트 |
| 1-C | 🔗 연쇄 영향 탐지 (도미노 체크) ※ 추후 개발 | 개정된 규정으로 인해 추가로 바꿔야 할 다른 규정 리스트 + 사유 |
| 3 | 🎯 경영 방향성 기반 전사 규정 일괄 정렬 | 일괄 개정안 + 신구조문대비표 |
| 4 | ⚖️ 내부 규정 간 충돌 탐지 | 충돌 지점 + 중재안 |
| 5 | 🤖 내/외부 통합 컴플라이언스 챗봇 | 규정+판례 엮은 실시간 행동 가이드 |

## 🎯 타겟 고객

규제 변화에 민감한 B2B 조직:
- 금융기관
- 제약회사
- 공공기관
- 대기업 법무/준법감시 부서

---

## 🏆 차별화 포인트

1. **MCP 아키텍처**: 국가법령정보센터 실시간 연동 (단순 프롬프트가 아님)
2. **End-to-End 자동화**: 모니터링 → 개정안 → 사유서까지 한 번에
3. **팀 역량 정합성**: B2B 사업/예산/규제/전략 경험 보유 팀의 강점 극대화

---

## 📦 MVP 범위

- **P0**: 기능 1-A, 1-B (디텍팅 → 선택 → 개정안 생성 + 판례 검증)
- **P0.5**: 기능 1-C (연쇄 영향 탐지 — 1-A, 1-B 완성 후 착수)
- **P1**: 기능 4, 5 (충돌 탐지 + 챗봇)
- **P2**: 기능 3 (경영 방향성 기반 일괄 정렬)

### MVP 데모 시나리오
1. 사내 규정 10개 세팅
2. "최근 법령 개정 스캔" 실행
3. → "취업규칙, 안전보건관리규정, 개인정보처리규정 3건 영향받음" 알림
4. 사용자가 "취업규칙" 선택
5. → 개정 사유 + 신구조문대비표 출력
