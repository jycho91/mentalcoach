# 마음 코치 — 기획안 v4

## 한 줄 요약
> "오늘 어땠는지 말해보세요. AI가 점점 더 깊이 나를 분석해줍니다. 쓸수록 정확해진다."

---

## 타겟
- 고민 많은 일반인 (20~30대 직장인 중심)
- 상담을 받을지 말지 고민하는 사람
- 특별한 문제가 없어도 자기 이해에 관심 있는 사람

**시장 근거:**
- 직장인 69%가 번아웃 경험 (잡코리아)
- 2026 Z세대 핵심 키워드 = "메타센싱" (감정을 객관적으로 감지·관리)
- 20대 중 24.5%가 AI 심리상담 경험 (대학내일20대연구소)

---

## 포지셔닝

| | 파인드마이와이 | 일반 감정일기앱 | AI 챗봇 (Woebot 등) | **마음코치** |
|--|--------------|---------------|-------------------|------------|
| 기록 방식 | 텍스트 답변 | 텍스트/이모지 | 대화형 체크인 | **음성으로 말하기** |
| 분석 깊이 | 깊음 (1회) | 없음 | 매일 동일 | **점점 깊어짐 (Lv.1→5)** |
| 기록+대화 연결 | 불가 | 기록만 | 대화만 | **기록이 대화·리포트에 반영** |
| 변화 추적 | 불가 | 캘린더 시각화 | 불가 | **매주 정량 비교** |
| 맥락 기억 | 1회성 | 없음 | 못함 (최대 불만) | **전체 기록 참조** |
| 사고 패턴 표현 | - | - | CBT 전문용어 | **일상 언어** |
| 가격 | 3~10만원 | 무료 | 유료 구독 | 무료 |

**핵심 메시지:** "타이핑 말고 그냥 말하세요. 쓸수록 나를 더 잘 아는 AI"

**차별화 3가지:**
1. **데이터 축적 → 분석 깊어짐** — 시장에 이 구조를 가진 앱이 없음 (블루오션)
2. **기록과 AI대화가 하나로 연결** — 감정일기앱은 기록만, AI챗봇은 대화만
3. **사고 패턴을 일상 언어로** — "파국화" 대신 "안 좋은 일에 최악을 먼저 떠올리는 패턴"

## 두 기능의 차별화

| | 🎙️ 마음 기록 | 💬 마음 대화 |
|--|------------|------------|
| 입력 방식 | **음성** (말하기) | 텍스트 (채팅) |
| 느낌 | 일기 쓰듯 혼자 털어놓기 | AI와 주고받기 |
| 목적 | 하루 감정 기록 → 패턴 분석 재료 | 실시간 공감/코칭 대화 |
| 분석 활용 | 주간 리포트 / 심층 분석 데이터 | 즉각적인 감정 반응 |

## 핵심 구조: 단계별 잠금 해제

```
가입 ──→ 초기 설문 ──→ Lv.1 기본 프로필
              │
      🎙️ 매일 음성 기록 / 💬 마음 대화
              │
         ┌────┴────┐
      3일째     7일째
    Lv.2        Lv.3
  표현 분석   첫 주간 리포트
              │
         ┌────┴────┐
   14일+대화3회  21일+대화5회
    Lv.4          Lv.5
  심화 리포트    ★ 심층 분석 ★
                    │
              (문제 감지 시)
                    │
            전문가 상담 안내
```

| 단계 | 조건 | 기능 |
|------|------|------|
| **Lv.1** | 가입 즉시 | 초기 프로필 리포트 |
| **Lv.2** | 기록 3일 | 자주 쓰는 표현 + 감정 어휘력 분석 |
| **Lv.3** | 기록 7일 | 첫 주간 리포트 + 감정 에너지 맵 |
| **Lv.4** | 기록 14일 + 마음 대화 3회 | 심화 리포트 (변화 카드 + 번아웃 체크) |
| **Lv.5** | 기록 21일 + 마음 대화 5회 | 심층 분석 (초기 프로필 비교 + 전문가 안내) |

**잠금 해제 UX:** 미해제 기능은 누르면 "기록 OO일 + 마음 대화 OO회 달성 시 이용 가능" + 진행률 바 표시. 숨기지 않고 보여줘서 동기 유발.

## ★ 분석 파이프라인 — "정량은 코드가, 해석은 LLM이"

### 아키텍처

```
[사용자 입력: 음성/텍스트]
         │
    ① JS 정량 계산 (API 없이 동작)
    ├─ 감정 단어 매칭 & 카운팅
    ├─ 극단어/의무어/전환어 빈도
    ├─ 문장 길이, 기록 빈도
    └─ 에너지/기분 좌표 산출
         │
    ② 정량 데이터 + 심리학 프레임 → 프롬프트 조합
         │
    ③ Gemini API가 "해석"만 담당
         │
    ④ 레벨별 다른 깊이의 리포트 출력
```

**왜 이렇게 나누나?**
- LLM에 "알아서 분석해줘"라고 하면 매번 결과가 다름
- JS가 숫자를 계산하면 → 일관된 기준 + 재현 가능
- LLM은 숫자를 사람이 이해하기 쉬운 말로 바꾸는 역할만

### 정량 지표 목록

#### A. 감정 어휘 분석 (Lv.2~)

| 지표 | 계산 방법 | 의미 |
|------|----------|------|
| 고유 감정 단어 수 | 감정 사전(약 80단어)과 매칭 후 중복 제거 | 감정 인식 폭 |
| 감정 다양성 비율 | 고유 감정 단어 / 전체 감정 단어 사용 횟수 | 1에 가까우면 다양, 0에 가까우면 반복 |
| 감정 카테고리 분포 | 기쁨/슬픔/분노/불안/평온 5분류 비율 | 어떤 감정에 치우쳐 있는지 |
| 주차별 변화량 | 이번 주 고유 단어 수 - 지난 주 | 성장(+) 또는 둔화(-) |

**감정 단어 사전 구조:**
```javascript
const EMOTION_DICT = {
  기쁨: ["행복", "즐겁", "뿌듯", "설레", "감사", "좋았", "신나", "만족", ...],
  슬픔: ["슬프", "우울", "외롭", "서운", "허전", "그립", "눈물", ...],
  분노: ["화나", "짜증", "억울", "답답", "열받", "분하", ...],
  불안: ["불안", "걱정", "두렵", "긴장", "조마조마", "초조", ...],
  평온: ["편안", "차분", "여유", "평화", "담담", "잔잔", ...]
};
```

#### B. 표현 패턴 분석 (Lv.2~)

| 지표 | 계산 방법 | 의미 |
|------|----------|------|
| 극단어 빈도 | "항상/절대/매번/절대로/전혀" 카운팅 | 흑백사고 경향 |
| 의무어 빈도 | "해야/해야지/해야만/~해야 한다" 카운팅 | 자기 압박 경향 |
| 전환어 빈도 | "근데/그래도/하지만/그런데도" 카운팅 | 리프레이밍 능력 (긍정 지표) |
| 자기비하 표현 | "나는 못/항상 이런 식/역시 안 돼" 카운팅 | 자책 경향 |
| 타인 귀인 표현 | "덕분에/~가 해줘서/~이 도와줘서" 카운팅 | 성과를 타인에게 돌리는 경향 |
| 감정 회피 표현 | "괜찮아/별거 아니야/그냥/뭐" 카운팅 | 감정 억제 경향 |
| 문장 평균 길이 | 총 글자 수 / 문장 수 | 길어지면 표현력↑, 짧아지면 에너지↓ |

#### C. 감정 에너지 맵 (Lv.3~)

2축 좌표 시스템:
```
        에너지 높음
           ↑
   흥분/스트레스 │ 활기/열정
  (High Energy  │ (High Energy
   Unpleasant)  │  Pleasant)
  ──────────────┼──────────────→ 기분 좋음
   무기력/우울   │ 평온/만족
  (Low Energy   │ (Low Energy
   Unpleasant)  │  Pleasant)
           ↓
        에너지 낮음
```

| 지표 | 계산 방법 |
|------|----------|
| 기분 점수 (-1~+1) | (긍정 감정 단어 수 - 부정 감정 단어 수) / 전체 감정 단어 수 |
| 에너지 점수 (-1~+1) | (감탄사 빈도 + 문장 길이 + 기록 길이) 정규화 |
| 요일별 평균 좌표 | 해당 요일 기록들의 평균 |
| 주차별 중심점 | 해당 주 전체 기록의 평균 좌표 |
| 분포 집중도 | 4사분면 중 어디에 몇 % 집중 |

#### D. 번아웃 체크 (Lv.4~)

5단계 모델:
```
1단계: 의욕 과잉기  — 의무어↑, 에너지↑, 긍정↑
2단계: 스트레스 축적기 — 극단어↑, 에너지 변동↑
3단계: 만성 피로기  — 에너지↓, 문장 길이↓, 감정 회피↑
4단계: 번아웃 진입기 — 부정↑, 자기비하↑, 감정 단어↓
5단계: 탈진기     — 기록 빈도↓, 응답 극단적으로 짧음
```

| 지표 | 계산 방법 | 단계 판단 기여 |
|------|----------|--------------|
| 주차별 에너지 평균 추이 | 주 평균 에너지 점수의 변화 방향 | 하락 추세 → 3단계+ |
| 부정 표현 증가율 | (이번 주 - 지난 주) / 지난 주 × 100 | 30%↑ → 2단계+ |
| 기록 빈도 변화 | 이번 주 기록 일수 / 지난 주 기록 일수 | 급감 → 4단계+ |
| 감정 회피 비율 | 감정 회피 표현 / 전체 문장 수 | 20%↑ → 3단계+ |
| 문장 길이 변화율 | 이번 주 평균 / 지난 주 평균 | 30%↓ → 4단계+ |

**판단 방식:** 각 지표에 가중치 부여 → 종합 점수로 5단계 중 하나 배정. LLM이 추가 맥락 해석.

## ★ 양방향 분석 설계 — 긍정 사용자도 커버

**문제:** 기존 설계는 부정적 패턴(자책, 번아웃) 감지에 치우쳐 있어서, 좋은 얘기만 하는 사용자에게 "할 말이 없는" 리포트가 됨.

**해결:** 모든 지표를 양방향으로 해석.

| 지표 | 부정 쪽 해석 | 긍정 쪽 해석 |
|------|------------|------------|
| 감정 어휘력 | "힘들다"만 반복 → 세분화 필요 | "좋다"만 반복 → 세분화 필요 |
| 극단어 | "항상 이런 식" → 흑백사고 | 극단어 적음 → 유연한 사고 (강점) |
| 전환어 | 없음 → 관점 전환 어려움 | 많음 → 리프레이밍 능력 높음 (강점) |
| 에너지 맵 | 에너지↓ 기분↓ → 번아웃 경고 | 기분↑ 에너지↓ → 과로 신호 |
| 의무어 | "해야 한다" 반복 → 자기 압박 | 적음 → 자율적 동기 (강점) |
| 타인 귀인 | - | "~덕분에"만 → 본인 역할 인정 연습 필요 |
| 감정 회피 | "괜찮아" 반복 → 감정 억제 | 감정 구체적 표현 → 감정 인식 높음 (강점) |
| 문장 길이 변화 | 짧아짐 → 에너지 저하 | 길어짐 → 표현력 성장 (강점) |

### 긍정 사용자 리포트 예시 (Lv.4)

```
📊 심화 리포트

💬 감정 어휘력: 12단어 → 18단어 (↑50%)
  → 감정을 점점 세밀하게 구분하고 있어요!

🗺️ 에너지 맵: 기분 좋음 + 에너지 낮음 영역에 62% 집중
  → 즐겁지만 체력적으로 무리하고 있을 수 있어요

🔄 변화 카드
  1주차: "오늘도 괜찮았다"
  3주차: "프레젠테이션이 잘 돼서 뿌듯했다"
  → 표현이 구체적으로 변하고 있어요!

🧠 발견된 표현 습관
  • "~해야지" 하루 평균 2.3회 — 자기에게 높은 기준을 두는 편
  • 성과를 타인 덕으로 돌리는 표현 5회 — 본인 역할도 인정해보세요

🔥 번아웃 체크: 1단계 (의욕 과잉기)
  █░░░░ 에너지가 높지만 지속 가능한지 체크해보세요

💡 맞춤 제안
  1. 오늘 "내가 잘한 일" 하나 구체적으로 말해보기
  2. "해야지" 대신 "하고 싶다"로 바꿔서 말해보기
  3. 쉬는 날 에너지 수준 관찰해보기
```

## ★ LLM 프롬프트 설계 — 전문가 관점 적용

각 리포트 생성 시 Gemini에 전달하는 프롬프트 구조:

```
[시스템 역할]
당신은 인지행동치료(CBT)와 긍정심리학에 기반한 심리 분석가입니다.
전문 용어를 쓰지 않고, 일상 언어로 설명합니다.
판단하지 않고, 발견한 패턴을 사용자가 스스로 인식하도록 돕습니다.

[사용자 프로필]
{초기 설문 결과 JSON}

[정량 데이터]
{JS가 계산한 지표 JSON}
- 감정 어휘: {고유 단어 수, 다양성 비율, 카테고리 분포}
- 표현 패턴: {극단어, 의무어, 전환어, 자기비하, 타인귀인, 감정회피 빈도}
- 에너지 맵: {주차별 중심점, 요일별 패턴, 분포 집중도}
- 번아웃 지표: {종합 점수, 단계}
- 변화 추이: {주차별 비교 데이터}

[원문 기록 (최근 N일)]
{사용자의 실제 기록 텍스트}

[요청]
위 정량 데이터를 바탕으로 {레벨별 리포트 양식}에 맞춰 해석해주세요.
- 숫자를 그대로 나열하지 말고, 사용자가 공감할 수 있는 스토리로 풀어주세요
- 강점을 먼저 언급하고, 개선 포인트는 제안 형태로 부드럽게
- 긍정적 변화가 있으면 반드시 짚어주세요
```

**전문가 관점 선택 (친구 피드백 반영):**
- 기본: CBT + 긍정심리학 프레임
- Lv.4~5에서는 번아웃 이론(Maslach) 프레임 추가
- 프롬프트에 프레임을 명시하여 → 일관된 관점의 분석 보장

## ★ 능동 대응 — 사용자 성향별 개인화 (친구 피드백 반영)

초기 설문 결과로 사용자를 성향 분류 → 이후 코칭/리포트가 달라짐:

| 성향 | 특징 | 코칭 방향 | 리포트 강조점 |
|------|------|----------|-------------|
| 내면화형 | 감정을 안으로 삼킴 | 감정 표현 연습 유도 | 감정 어휘력 성장 부각 |
| 외현화형 | 감정을 바로 표출 | 감정 조절 전략 제안 | 에너지 맵 안정화 변화 |
| 회피형 | 감정 자체를 무시 | "괜찮아"가 진짜인지 질문 | 감정 회피 패턴 인식 |
| 분석형 | 이성적으로 접근 | 감정에 이름 붙이기 | 감정 vs 생각 구분 |
| 완벽주의형 | 높은 기준, 자기비판 | "해야 한다" → "하고 싶다" 전환 | 의무어 변화 추적 |

**구현:** 초기 설문 답변 조합으로 성향 태그 부여 → 프롬프트에 성향 정보 포함

## 심리 분석 기법 상세

### 1. 감정 에너지 맵
감정을 "에너지 높음/낮음" × "기분 좋음/나쁨" 2축으로 매핑.
Russell의 Circumplex Model of Affect 기반.
매주 점이 어디에 찍히는지 보면 변화가 한눈에 보임.
- 적용: Lv.3부터
- 정량: 기분 점수, 에너지 점수 (JS 계산)
- 해석: 분포 패턴, 이동 방향 (LLM 해석)

### 2. 감정 어휘력 분석
"힘들다"만 반복하는지, "억울하다/서운하다"를 구분해서 쓰는지.
단어 수 증가 자체가 성장의 증거. 긍정/부정 모두 적용.
- 적용: Lv.2부터
- 정량: 고유 단어 수, 다양성 비율, 카테고리 분포 (JS 계산)
- 해석: 성장 의미, 개선 방향 (LLM 해석)

### 3. 번아웃 단계 체크 (5단계)
Maslach Burnout Inventory 기반 5단계 모델 적용.
기록 변화 추이로 현재 번아웃 단계 추정.
타겟(직장인)에게 직접적으로 와닿음.
- 적용: Lv.4부터
- 정량: 에너지 추이, 부정 표현 증가율, 기록 빈도 변화 등 (JS 계산)
- 해석: 현재 단계 설명, 관리 전략 (LLM 해석)

### 4. 사고 패턴 감지 (일상 언어 버전)
CBT의 인지왜곡 개념을 일상 언어로 풀어서 전달.
- 파국화 → "안 좋은 일에 최악을 먼저 떠올리는 패턴"
- 당위 진술 → "~해야 한다는 생각"
- 개인화 → "다 내 탓이라고 생각하는 패턴"
- 타인 귀인 → "좋은 결과를 운이나 다른 사람 덕으로 돌리는 패턴"
- 적용: Lv.4~5
- 정량: 극단어, 의무어, 자기비하, 타인귀인 빈도 (JS 계산)
- 해석: 패턴 명명 + 일상 언어 설명 + 원문 예시 인용 (LLM 해석)
