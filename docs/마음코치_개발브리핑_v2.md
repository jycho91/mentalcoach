이 문서는 프로젝트의 전체 맥락, 기획 의도, 기술 요구사항을 담고 있다. 이 파일 하나만 읽으면 프로젝트를 완전히 이해할 수 있도록 작성되었다.

## 파트 B: 기획 의사결정 히스토리

이 기획은 여러 차례 피드백과 개선을 거쳐 도출되었다. 아래는 주요 의사결정의 흐름이다.

### 초기 기획 (팀원 제안)
- "마음 코치 마스터 기획서 V4.0" — SOAP 노트 자동화, 위기 감지, 헬스킷 연동, Voice-to-Voice, 듀얼 대시보드, B2C Freemium 등 10개 이상의 기능을 담은 대형 기획
- 문제점: 기능이 너무 많아 초점이 없고, MVP로 구현 불가능. "다 합친다"는 전략. AI-Native 논리가 약함 (각 기능은 이미 개별 서비스로 존재)

### 1차 개선: 기능 축소
- 핵심을 "세션 브리핑 한 장"으로 축소
- 167시간(세션 밖 시간)의 데이터를 모아서, 다음 세션 전에 상담사가 3분 만에 파악하는 AI 브리핑
- 팀원의 실제 상담 경험("매번 지난주 이야기를 처음부터 설명하는 게 시간 낭비")이 Why You의 근거

### 2차 개선: 차별화 강화
- "편파적 브리핑(Stance-Aware Summarization)" 추가: 같은 데이터를 내담자에게는 따뜻하게, 상담사에게는 임상적으로 변환
- 세션 연속성 타임라인: 3~4주 흐름을 종단으로 시각화
- 세션 어젠다 제안: AI가 이번 세션에서 다룰 주제 우선순위 제안

### 3차 개선: 범용성 확보
- 상담 안 받는 일반인도 쓸 수 있는 제품으로 확장
- "사고 패턴 미러링" 추가: 기록에서 반복되는 사고 습관을 AI가 감지해서 알려줌
- 온보딩에서 "상담 받고 계세요?" 분기 → 일반 모드 / 상담 모드
- AI 대화 모드 추가: 공감, 코칭, 응원, 오늘의 질문

### 4차 개선: 기존 초안 기능 통합
- 팀원이 만든 기존 HTML 초안의 과제 체크인 기능을 상담 모드에 통합
- 별도 탭이 아니라 기록 화면에 자연스럽게 녹임
- 통계/히스토리 탭은 제거 → 주간 리포트와 타임라인이 대체
- AI 피드백은 매일 격려가 아니라 주간 리포트에 통합

### 5차 개선: 리스크 방어
- 유저 대면 화면에서 임상 용어(인지 왜곡, 파국화, 흑백사고) 제거 → 일상 표현으로 대체
- 사주/타로 기능 제거 → "오늘의 질문" (심리학 기반 리플렉션 질문)으로 교체하여 앱 정체성 일관성 확보
- 상담사 브리핑에 "AI 생성 참고자료" 디스클레이머 필수 표시
- 정량 데이터(이행률, 빈도)는 코드에서 직접 계산, LLM은 해석만 담당하여 할루시네이션 리스크 분리

## 파트 C: 최종 기획안

### 한 줄 요약
"매일의 감정을 기록하고 AI와 대화하며 자기 사고 습관을 알아차리게 해주는 앱. 상담 받는 사람에게는 과제 추적과 세션 전 맞춤형 브리핑까지 만들어주는 확장형 구조."

### 온보딩
첫 실행 시 하나만 묻는다: "현재 상담을 받고 계세요?"
- "아니요" → 일반 모드
- "네" → 상담 모드 (일반 모드 전체 + 상담 연동 기능)
- 설정에서 언제든 전환 가능

---

### 일반 모드 (모든 사용자) — 3개 화면

**화면 1: 오늘 기록**
- 자유 텍스트 영역: "오늘 어땠어요?" — 한 줄~세 줄
- 음성 입력 버튼 (Web Speech API 활용, 음성→텍스트 변환)
- 이 화면이 매일 앱을 여는 진입점

**화면 2: AI 대화**
- 모드 선택 UI (카드 또는 탭):
  - 공감 모드: 따뜻하게 들어주고 감정 반영. 해결책 제시 안 함.
  - 코칭 모드: "다르게 볼 수 있을까?" 리프레이밍 질문.
  - 응원 모드: 무조건 긍정 에너지.
  - 오늘의 질문: 매일 하나씩 심리학 기반 리플렉션 질문 제공. (예: "오늘 하루 중 가장 감사했던 순간은?", "최근 나를 웃게 만든 작은 일은?", "지금 가장 기대되는 일이 있다면?") 사주/타로가 아닌, 앱의 정체성과 일관된 자기 성찰 질문.
- 채팅 UI (메시지 주고받기)
- 모든 대화 내용은 저장되어 사고 패턴 분석 재료가 됨

**화면 3: 주간 리포트**
- AI가 1주일 기록(자유 텍스트 + 대화 내용)을 분석하여 생성
- **중요: 유저에게는 임상 전문 용어를 사용하지 않는다**
- 구성 요소:
  - 주간 감정 흐름 요약
  - 자주 쓴 표현 하이라이트
  - 사고 습관 패턴 알림 (반드시 일상 언어로):
    - ❌ "파국화 인지 왜곡 3회 감지"
    - ⭕ "안 좋은 일이 생겼을 때 '최악의 결과'를 먼저 떠올리는 패턴이 3번 있었어요"
    - ❌ "흑백사고 패턴"
    - ⭕ "'항상', '절대' 같은 전부-아니면-전무 표현이 자주 보여요"
    - ❌ "과일반화"
    - ⭕ "한 번 있었던 일을 '매번 그래'로 넓혀서 생각하는 경향이 있었어요"
  - 부드러운 리프레이밍 제안
  - 감정 흐름 시각화 (간단한 차트)

---

### 상담 모드 (상담 받는 사용자) — 5개 화면

일반 모드의 3개 화면에 아래가 추가/확장됨.

**화면 1 확장: 오늘 기록 + 과제 체크인**
- (상단) 자유 텍스트: "오늘 어땠어요?" — 일반 모드와 동일
- (하단) 오늘의 과제 체크리스트:
  - 각 과제별 완료 ✓ / 미완료 ✗ 버튼
  - 미완료 시 "어떤 점이 어려웠나요?" 한 줄 메모 입력 (선택)
  - 과제 등록/삭제는 별도 설정에서 관리

**화면 3 확장: 주간 리포트 + 과제 이행 패턴**
- 일반 모드 리포트 내용 + 아래 추가:
  - 과제별 이행률: "산책 10분 2/5일, 감정 일기 4/5일" (코드에서 직접 계산)
  - 이행 패턴과 감정 교차 분석 (AI 해석): "과제를 못 한 날에 자책하는 표현이 집중되어 있어요"

**화면 4: 세션 준비 (내담자/상담사 토글)**
- 상단에 토글 스위치: [내담자 뷰 🌱] / [상담사 뷰 🩺]
- [브리핑 생성하기] 버튼
- 버튼 누르면 LLM API 2회 호출 (내담자용 + 상담사용 동시 생성)
- 토글 전환 시 같은 카드 영역에 내용만 바뀜 (부드러운 페이드 전환)

내담자 뷰 (따뜻한 톤, 임상 용어 사용하지 않음):
  - "이번 주 이런 일이 있었어요"
  - "다음 세션에서 이 이야기를 꺼내보면 어떨까요?"
  - 과제 이행 요약 (격려 톤)

상담사 뷰 (임상 톤, 전문 용어 사용):
  - 정량 데이터 섹션 (코드 계산, 할루시네이션 없음):
    - 과제별 이행률 수치
    - 특정 표현 빈도 횟수
  - AI 해석 섹션 (LLM 생성, 시각적으로 구분 표시):
    - 감정 패턴 해석
    - 세션 어젠다 제안
  - 하단 디스클레이머 (필수):
    "⚠️ 본 브리핑은 AI가 생성한 참고자료이며, 임상적 판단을 대체하지 않습니다. 최종 판단은 상담사에게 있습니다."

**화면 5: 타임라인 (상담 모드 전용)**
- 세로 타임라인 UI
- 각 주차별 노드: 날짜 + 핵심 주제 + 감정 키워드 + 과제 이행률
- 노드 사이에 연결선
- 가장 최근이 상단, 아래로 스크롤하면 과거

### 네비게이션
- 하단 탭 바 (모바일 네이티브 느낌)
- 일반 모드: 3개 탭 (기록 / 대화 / 리포트)
- 상담 모드: 5개 탭 (기록 / 대화 / 리포트 / 세션 준비 / 타임라인)
- 우측 상단: 설정 기어 아이콘

### 설정 화면 (모달)
- 모드 전환: 일반 ↔ 상담
- [상담 모드] 과제 관리: 추가/삭제
- API 키 입력란
- 다크 모드 토글
- 데이터 내보내기/가져오기 (JSON)
- 데모 데이터 불러오기 버튼
- 데이터 초기화 (위험 구역)

## 파트 D: 기술 요구사항

### 기술 스택 (바이브코딩 친화적)
- **프론트엔드**: HTML + CSS + JavaScript (단일 파일 또는 소수 파일)
  - React를 쓸 수 있지만, 팀원이 관리 가능한 수준으로 유지
  - 모바일 반응형 필수 (데모를 폰으로 보여줄 수 있어야 함)
- **데이터 저장**: localStorage 또는 IndexedDB
  - 서버/DB 없이 브라우저에서 모든 데이터 관리
  - 데모용이므로 데이터 영속성은 중요하지 않음
  - JSON 내보내기/가져오기로 백업 지원
- **AI 연동**: Anthropic Claude API (claude-sonnet-4-20250514)
  - API 키는 환경변수 또는 설정 화면에서 입력받음
  - 비용 절약을 위해 Sonnet 모델 사용 (Opus 대비 저렴)
- **배포**: GitHub Pages, Vercel, 또는 Netlify (무료 티어)
- **음성 입력**: Web Speech API (브라우저 내장, 무료)
  - SpeechRecognition API로 음성→텍스트 변환
  - 별도 API 비용 없음

### API 비용 최소화 전략
- Claude Sonnet 사용 (입력 $3/M tokens, 출력 $15/M tokens)
- 대화 1회당 약 1,000~2,000 토큰 → 약 20~50원
- 주간 리포트 1회당 약 3,000~5,000 토큰 → 약 100~200원
- 세션 브리핑 생성 시 2회 호출 (내담자용 + 상담사용) → 약 200~400원
- 하루 테스트 10회 기준 약 500원, 한 달 약 15,000원
- 데모데이까지 총 예상 비용: 3~5만원 (10만원 예산 내 충분)
- 추가 절약: AI 대화에서 대화 히스토리 길이를 최근 10턴으로 제한

### LLM API 호출 구조

**1. AI 대화 (실시간)**
```
POST /v1/messages
{
  model: "claude-sonnet-4-20250514",
  max_tokens: 500,
  system: [모드별 시스템 프롬프트 — 아래 참조],
  messages: [최근 10턴 대화 히스토리]
}
```

**2. 주간 리포트 생성**
```
POST /v1/messages
{
  model: "claude-sonnet-4-20250514",
  max_tokens: 1500,
  system: "당신은 사용자의 감정 기록을 분석하는 따뜻한 코치입니다. 사용자의 1주일간 기록을 분석하여 반복되는 사고 습관을 감지하고, 일상적인 언어로 부드럽게 알려주세요. 절대로 '인지 왜곡', '파국화', '흑백사고' 같은 임상 전문 용어를 사용하지 마세요. 대신 구체적인 표현 예시를 들어 '이런 습관이 보여요'라고 설명하세요. 이것은 진단이 아니라 자기 인식을 돕는 것입니다.",
  messages: [{
    role: "user",
    content: "아래는 사용자의 최근 1주일 기록입니다:\n\n[1주일치 기록 데이터 JSON]\n\n이 기록을 분석하여 다음을 JSON 형태로 제공하세요:\n1. 주간 감정 흐름 요약 (3~4문장, 따뜻한 톤)\n2. 자주 쓴 표현과 그 빈도\n3. 반복되는 사고 습관 패턴과 해당 표현 예시 (일상 언어로, 전문 용어 금지)\n4. 리프레이밍 제안 (2~3개, 부드러운 질문 형태)\n5. 감정 키워드별 빈도 (차트용 데이터)"
  }]
}
```

**3. 세션 브리핑 — 내담자용**
```
POST /v1/messages
{
```

### 정량 데이터 계산 함수 (코드에서 직접 계산, LLM 아님)

- 과제별 이행률: 각 과제가 지난 7일간 완료된 일수 / 기록된 일수
- 전체 이행률: 전체 완료 수 / 전체 과제 수
- 특정 표현 빈도: 기록 텍스트에서 특정 키워드("항상", "절대", "또", "어차피", "못", "안 돼" 등) 출현 횟수

## 파트 H: 주의사항

1. **유저 대면 언어 규칙**: 내담자/일반 유저에게 보이는 모든 텍스트에서 임상 전문 용어를 사용하지 않는다. "인지 왜곡" → "사고 습관", "파국화" → "최악을 먼저 떠올리는 패턴", "흑백사고" → "전부-아니면-전무 표현". 상담사 뷰에서만 전문 용어를 사용한다.

2. **디스클레이머 필수**: 상담사 뷰 브리핑 하단에 항상 "AI 생성 참고자료" 디스클레이머를 표시한다.

3. **정량/해석 분리**: 상담사 뷰에서 코드로 계산한 정량 데이터와 LLM이 생성한 해석을 시각적으로 구분한다.

4. **API 키 보안**: API 키를 코드에 하드코딩하지 말 것. 설정 화면에서 사용자가 직접 입력하고 localStorage에 저장.

5. **에러 핸들링**: API 호출 실패 시 사용자 친화적 에러 메시지 표시. 앱이 멈추지 않게 할 것.

6. **데모용 더미 데이터**: 설정에서 [데모 데이터 불러오기] 버튼으로, 월~금 5일치 기록 + 과제 체크인 + AI 대화 히스토리가 자동으로 채워지게 할 것. 더미 데이터는 반복되는 사고 패턴("항상", "또", "어차피" 등)이 명확히 드러나는 내용으로 작성.

7. **반응형**: 노트북과 폰 양쪽에서 깨지지 않게 만들 것.

8. **로딩 UX**: LLM API 호출 중 예쁜 로딩 애니메이션 필수. "AI가 분석 중..." 느낌.

9. **CORS**: 브라우저에서 Anthropic API 직접 호출 시 CORS 문제 가능. 간단한 프록시(Vercel Serverless Function 등) 또는 가능한 가장 간단한 방법 선택.

10. **오늘의 질문에서 사주/타로/운세 표현 금지**: "오늘의 질문" 모드에서 점술적 표현을 사용하지 않는다. 자기 성찰과 마음 돌봄 톤을 유지한다.
