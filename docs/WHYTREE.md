# WHYTREE.md — 마음 코치 Why Tree

## 출발점

감정 기록

## Why Up (왜?)

```
감정 기록
    │ why?
나를 돌아보기
    │ why?
자기 이해
    │ why?
마음의 평안
    │ why?
좋은 하루
```

## How Down (어떻게?)

"좋은 하루"는 너무 넓다. 한 단계 내려와서 **"자기 이해"**에서 How Down.

```
자기 이해
├── 나를 돌아보기
│   ├── 감정 기록
│   ├── 대화 (누군가에게 말하기)
│   └── 혼자 생각하기
│
├── 내 패턴 알아차리기
│   ├── 반복되는 생각 인식
│   ├── 감정 흐름 파악
│   └── 변화 비교
│
└── 더 나은 나
    ├── 다른 관점 갖기
    ├── 작은 성장 확인
    └── 필요하면 도움 요청
```

## Why Up ↔ How Down 교차

How Down에서 나온 "내 패턴 알아차리기"를 다시 Why Up.

```
내 패턴 알아차리기
    │ why?
같은 고민을 반복하고 싶지 않아서
    │ why?
자기 이해 ← (다시 수렴)
```

How Down에서 나온 "변화 비교"를 다시 Why Up.

```
변화 비교
    │ why?
나아지고 있는지 알고 싶어서
    │ why?
작은 성장이라도 확인하고 싶어서
    │ why?
마음의 평안 ← (다시 수렴)
```

## Why Tree에서 발견한 것

1. **"나를 돌아보기"와 "내 패턴 알아차리기"는 다른 HOW다.** 일기를 쓰는 건 돌아보기지만, 거기서 패턴을 발견하는 건 별개의 일이다. 사람은 혼자서 돌아보기는 할 수 있어도, 자기 패턴을 알아차리기는 어렵다. → 여기에 AI의 객관적인 분석이 필요하다.

2. **"변화 비교"가 독립적 가치다.** 기록을 쌓아야만 가능한 일이고, 하루짜리 분석으로는 불가능하다. → "쓸수록 나에 대한 이해가 깊어지는" 레벨 구조가 여기서 나왔다.

3. **"더 나은 나"의 HOW 중 "필요하면 도움 요청"이 마지막에 있다.** 처음부터 전문가를 찾는 게 아니라, 스스로 돌아보고 패턴을 인식한 뒤에 필요하면 가는 흐름이 자연스럽다. → 전문가 상담 권유를 레벨 후반에 배치한 이유.

이 Why Tree 작업을 통해, 단순한 감정 일기 앱이 아니라 **"기록 → 패턴 인식 → 변화 확인 → 필요 시 전문가 연결"**이라는 단계적 구조를 가진 AI 마음 코치를 기획하게 되었다.
