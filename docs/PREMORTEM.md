# PREMORTEM.md — 마음 코치 사전 부검

## 프리모템이란?

프로젝트가 실패했다고 가정하고, "왜 실패했는가"를 미리 상상하는 기법. 낙관적 편향을 깨고 현실적 리스크를 사전에 파악하기 위함.

---

## 시나리오: "마음 코치는 데모데이에서 낮은 평가를 받았다"

왜 실패했는가? 아래 리스크들이 실현되었기 때문이다.

---

### 리스크 1: 유저가 기록을 안 남긴다

**실패 시나리오:** 앱을 깔고 초기 설문은 했지만, 다음 날부터 안 들어온다. 매일 기록하는 건 귀찮고, 3일 뒤에 기록이 없으면 Lv.2 분석도 못 만든다. 핵심 가치인 "쓸수록 깊어지는 분석"이 작동하지 않는다.

**발생 확률:** 높음. 감정 일기 앱의 7일 리텐션은 업계 평균 10% 미만.

**방어책:**
- 음성 입력으로 마찰을 최소화 (말하기는 타이핑보다 10배 쉽다)
- 감정 태그 원터치로 "한 글자도 안 써도 데이터가 쌓이는" 구조
- 개인화 질문 카드로 "뭘 쓸지 모르겠다"를 해결
- 3일 이상 안 들어오면 복귀 질문으로 부드럽게 유도

---

### 리스크 2: AI가 잘못된 패턴을 감지한다 (할루시네이션)

**실패 시나리오:** 유저가 "오늘 친구랑 재밌게 놀았다"라고 썼는데, AI가 "자책 패턴이 감지되었습니다"라고 엉뚱한 분석을 내놓는다. 신뢰가 즉시 무너진다.

**발생 확률:** 중간. LLM은 본질적으로 할루시네이션 가능성이 있다.

**방어책:**
- 정량 데이터(표현 빈도, 감정 태그 통계)는 코드에서 직접 계산 → 할루시네이션 없음
- LLM은 "해석"만 담당하고, 해석 섹션은 시각적으로 구분 표시
- 상담사 뷰에는 "AI 생성 참고자료" 디스클레이머 필수 표시
- 프롬프트에 "유저가 직접 쓴 표현을 근거로만 분석하세요. 추측하지 마세요." 명시

---

### 리스크 3: "의료 행위" 논란

**실패 시나리오:** 교수님이 "이거 의료 행위 아닌가요? 진단을 내리는 건 불법인데"라고 질문한다. 답변을 못 하면 프로젝트의 신뢰도가 무너진다.

**발생 확률:** 중간. 멘탈케어 서비스에서 항상 나오는 질문.

**방어책:**
- 유저 대면 화면에서 임상 전문 용어(인지 왜곡, 파국화 등) 사용 금지
- "진단"이 아니라 "패턴 관찰"로 일관된 포지셔닝
- "당신은 OO입니다"가 아니라 "이런 표현이 반복되고 있어요"
- 상담 권유도 "치료를 받으세요"가 아니라 "관심 있으면 전문가와 이야기해보는 것도 방법이에요"

---

### 리스크 4: 음성 입력이 데모에서 실패한다

**실패 시나리오:** 발표장 소음 때문에 음성 인식이 안 되거나, 브라우저가 Web Speech API를 지원하지 않아서 마이크 버튼이 작동하지 않는다.

**발생 확률:** 중간. Web Speech API는 Chrome에서만 안정적으로 작동.

**방어책:**
- 데모 전에 발표용 노트북에서 Chrome으로 사전 테스트 필수
- 음성 입력 실패 시 텍스트 직접 입력으로 즉시 전환 가능한 UI
- 최악의 경우를 대비해 미리 입력된 더미 데이터로 시연 가능

---

### 리스크 5: API 호출이 데모 중 실패한다

**실패 시나리오:** 발표장 와이파이가 느리거나, API 서버가 일시적으로 응답하지 않아 리포트 생성이 멈춘다.

**발생 확률:** 낮음~중간. 네트워크 의존 이슈.

**방어책:**
- 테스트 모드로 API 없이도 모든 기능이 더미 응답으로 작동
- 핵심 시연 장면(초기 프로필, 주간 리포트, 세션 브리핑)은 사전에 생성해두고 캐싱
- 데모 시나리오에서 "실시간 API 호출이 필요한 순간"을 최소화 (초기 프로필 1회만)

---

### 리스크 6: 레벨 시스템이 데모에서 안 보인다

**실패 시나리오:** 데모는 5분인데, Lv.1~5를 다 보여주려면 21일치 데이터가 필요하다. 실제로 쌓을 수 없으니 더미 데이터에 의존하는데, "이거 진짜 되는 건가?"라는 의심을 받는다.

**발생 확률:** 중간. 시간 기반 기능의 근본적 한계.

**방어책:**
- 더미 데이터를 3주치로 정교하게 구성하여 Lv.4~5까지 시연 가능
- 초기 프로필(Lv.1)은 발표장에서 실시간 API로 시연하여 "진짜 된다"를 증명
- 발표 스크립트에서 "기록이 쌓이면 이렇게 깊어집니다"라고 시간 흐름을 내러티브로 설명

---

### 리스크 7: "긍정적인 사람은 분석할 게 없지 않나?"

**실패 시나리오:** 교수님이나 청중이 "나는 별 문제 없는데, 이 앱 왜 써야 하나?"라고 물었을 때 답을 못 한다.

**발생 확률:** 높음. So What 평가와 직결.

**방어책:**
- "좋다"만 반복하는 사람 → 감정 어휘가 단조롭다는 것 자체가 인사이트
- 기분↑ 에너지↓ → 과로 신호. 본인은 "괜찮다"고 느끼지만 데이터는 다르게 말한다
- "해야지" 같은 의무감 표현, "괜찮아" 같은 감정 회피 패턴은 긍정적인 사람에게도 있다
- 핵심 메시지: "힘든 사람에게는 패턴 인식, 괜찮은 사람에게는 강점 발견 + 숨은 피로 감지"

---

### 리스크 8: 앱 정체성 혼란

**실패 시나리오:** "이 앱이 일기장인지, 상담 앱인지, 분석 도구인지, 게임화 앱인지 모르겠다"는 피드백.

**발생 확률:** 중간. 기능이 여러 개 합쳐져 있으므로.

**방어책:**
- 발표에서 한 문장으로 정의: "매일 말하면, AI가 점점 더 깊이 나를 분석해주는 앱"
- 화면 4개(기록/대화/리포트/설정)로 단순하게 유지
- 레벨 시스템이 모든 기능을 하나의 내러티브("쓸수록 깊어진다")로 묶어주는 역할

---

## 요약: 리스크 매트릭스

| 리스크 | 확률 | 영향도 | 방어 상태 |
|--------|------|--------|----------|
| 기록 이탈 | 높음 | 치명적 | 음성+태그+질문카드로 3중 방어 |
| 할루시네이션 | 중간 | 높음 | 정량/해석 분리 + 디스클레이머 |
| 의료 행위 논란 | 중간 | 높음 | 용어 정책 + 포지셔닝 일관성 |
| 음성 실패 | 중간 | 중간 | 텍스트 폴백 + 사전 테스트 |
| API 실패 | 낮음~중간 | 높음 | 테스트 모드 + 캐싱 |
| 레벨 시연 한계 | 중간 | 중간 | 더미 데이터 + 내러티브 |
| 긍정인 분석 | 높음 | 중간 | 감정 어휘력 + 과로 신호 + 감정 회피 |
| 정체성 혼란 | 중간 | 중간 | 한 문장 정의 + 레벨로 통합 |
