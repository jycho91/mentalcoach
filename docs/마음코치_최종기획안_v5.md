# 마음 코치 v5 — 최종 기획안

## 한 줄 요약

"매일의 감정을 기록하고 AI와 대화하며 자기 사고 습관을 알아차리게 해주는 앱. 상담 받는 사람에게는 과제 추적과 세션 전 맞춤형 브리핑까지 만들어주는 확장형 구조."

---

## 1. 문제 정의

### 일반인의 문제

감정을 글로 쓰는 사람은 많지만, 자기가 어떤 사고 습관에 빠져 있는지 알아차리는 사람은 거의 없다. "또 망쳤다", "나는 항상 이래", "어차피 안 돼"를 반복하고 있어도 스스로는 모른다. 감정 일기장은 많지만, "네 일기에서 이런 패턴이 보여"라고 말해주는 일기장은 없다.

### 상담 받는 사람의 추가 문제

심리상담은 주 1회 50분이다. 나머지 167시간은 방치된다. 세션에 가면 "이번 주 어떠셨어요?"라는 질문에 1주일을 요약하기 어렵고, 15분을 근황 설명에 소비한다. 상담사가 준 과제(산책 10분, 감정 일기 쓰기 등)도 했는지 안 했는지 기억이 흐릿하다. 상담사 역시 하루 5~8명을 보면서 지난 세션 내용이 흐릿해서 준비에 10~15분, 일지 작성에 2시간을 추가로 쓴다.

### 기존 서비스의 한계

- B2C AI 챗봇 (Wysa, Woebot 등): 대화는 하지만 데이터가 상담사에게 전달 안 됨
- 매칭 플랫폼 (마인드카페 등): 연결만 하고 세션 밖은 방치
- 상담실 내 AI (Upheal 등): 세션 중 녹음만 정리, 세션 밖 일상은 추적 못 함
- 일반 감정 일기 앱: 기록만 쌓이고, 패턴 인식이나 피드백 없음
- 습관 트래커 앱: 체크만 하고 "왜 못 했는지"의 맥락은 없음

## 2. 제품 구조

### 온보딩 분기

첫 실행 시 하나만 묻는다: "현재 상담을 받고 계세요?"

- "아니요" → 일반 모드
- "네" → 상담 모드 (일반 모드 전체 + 상담 연동 기능)
- 설정에서 언제든 전환 가능

## 3. 일반 모드 (모든 사용자)

### 3-1. 매일 기록 화면

하나의 화면에서 그날의 기록을 모두 완료한다. 탭을 여러 개 오가지 않는다.

구성:
- 자유 텍스트 영역: "오늘 어땠어요?" — 한 줄~세 줄 (음성 입력 지원)
- 이 화면이 매일 앱을 여는 진입점이 된다

### 3-2. AI 대화 — 모드 선택형

사용자가 그날 필요에 따라 대화 모드를 선택한다.

**진지한 축 (코어):**
- 공감 모드: 따뜻하게 들어주고 감정을 반영. 해결책을 제시하지 않고 경청에 집중.
- 코칭 모드: "이 상황을 다르게 볼 수 있을까?" 식의 리프레이밍 제안. 사고를 전환하도록 질문을 던져준다.

**가벼운 축 (유입 + 재방문 유도):**
- 응원 모드: 무조건 긍정적인 에너지로 하루를 응원.
- 오늘의 질문: 매일 하나씩 심리학 기반 리플렉션 질문을 제공. (예: "오늘 하루 중 가장 감사했던 순간은?", "최근 나를 웃게 만든 작은 일은?", "지금 가장 기대되는 일이 있다면?") 가볍지만 앱의 정체성과 일관된 톤을 유지한다.

진지한 축이 메인이고, 가벼운 축은 카드형으로 부담 없이 접근 가능하게 배치한다. 모든 대화 내용은 기록 데이터에 축적되어 사고 패턴 분석의 재료가 된다.

### 3-3. 사고 패턴 미러링 — 핵심 차별화

매일 쌓인 기록(텍스트 + 대화)에서 AI가 사용자의 사고 습관을 읽어 거울처럼 비춰준다.

**중요: 유저에게는 임상 용어를 사용하지 않는다.** "인지 왜곡", "파국화", "흑백사고" 같은 전문 용어 대신 일상 표현을 사용한다. 이 앱은 진단 도구가 아니라 자기 인식 도구이다.

**주간 리포트 구성:**
- 자주 쓴 표현 하이라이트: "'항상', '또', '어차피' 같은 표현이 이번 주 5번 나왔어요"
- 사고 습관 패턴 알림 (일상 언어로):
  - ❌ "파국화 인지 왜곡 3회 감지" → ⭕ "안 좋은 일이 생겼을 때 '최악의 결과'를 먼저 떠올리는 패턴이 3번 있었어요"
  - ❌ "흑백사고 패턴" → ⭕ "'항상', '절대' 같은 전부-아니면-전무 표현이 자주 보여요"
  - ❌ "과일반화" → ⭕ "한 번 있었던 일을 '매번 그래'로 넓혀서 생각하는 경향이 있었어요"
- 부드러운 리프레이밍 제안: "정말 '항상' 그런 건지, 한 번 돌아보면 어떨까요?"
- 주간 감정 흐름 시각화: 어떤 요일에 어떤 감정이 많았는지

**왜 차별화인가:** 감정 일기 앱은 많지만, 기록에서 반복되는 사고 습관을 자동으로 감지해서 일상 언어로 알려주는 앱은 없다. LLM이 비정형 텍스트를 읽고 맥락을 이해해야 가능한 기능이다.

## 4. 상담 모드 (상담 받는 사용자)

일반 모드의 모든 기능에 더해서 아래 기능이 추가된다.

### 4-1. 매일 기록 화면 확장 — 과제 체크인 통합

일반 모드의 자유 기록 영역 아래에, 상담사가 준 과제 체크리스트가 자연스럽게 붙는다. 별도 탭이 아니라 하나의 기록 화면에 통합된다.

구성:
- (상단) 자유 텍스트: "오늘 어땠어요?" — 일반 모드와 동일
- (하단) 오늘의 과제 체크리스트:
  - 각 과제별 완료/미완료 체크
  - 못 한 과제에는 "어떤 점이 어려웠나요?" 한 줄 메모 (선택)
  - 이 "왜 못 했는지"의 맥락이 브리핑의 핵심 재료가 된다

과제 등록/삭제는 설정 또는 상담 기록 화면에서 간단히 관리한다. 체크인 화면은 기록에만 집중한다.

### 4-2. 주간 리포트 확장 — 과제 이행 패턴 통합

일반 모드의 사고 패턴 리포트에 과제 데이터가 합쳐진다. 별도 통계 탭을 만들지 않고, 주간 리포트 한 장에 모든 인사이트가 들어간다.

추가되는 내용:
- 과제 이행률: "이번 주 산책 10분 2/5일, 감정 일기 4/5일"
- 이행 패턴과 감정의 교차 분석: "과제를 못 한 날에 자책하는 표현이 집중되어 있어요. 못 한 것 자체보다 자책하는 습관을 다음 세션에서 이야기해보면 어떨까요?"

→ 단순 완료율 차트(습관 앱의 문법)가 아니라, "왜 못 했는지"와 "그때 어떤 감정이었는지"를 연결하는 것이 마음 코치의 차별점이다.

### 4-3. 세션 준비 브리핑 (내담자용)

다음 세션 전날, 1주일간 기록(자유 텍스트 + 과제 체크인 + AI 대화 내용)을 바탕으로 AI가 따뜻한 톤의 브리핑을 생성한다.

- "이번 주 이런 일이 있었어요" (주간 감정 흐름 요약)
- "이 부분을 다음 세션에서 꺼내보면 어떨까요?" (대화 주제 제안 2~3개)
- "수요일에 산책 과제를 못 했을 때 자책감을 느꼈다고 적으셨는데, 이 패턴을 상담사에게 이야기해보세요"
- 과제 이행 요약: "이번 주 전체 달성률 60%. 감정 일기는 잘 했지만 산책이 어려웠어요"

→ 내담자가 세션을 수동적으로 받는 게 아니라 주도적으로 준비할 수 있게 해준다.

### 4-4. 상담사 대시보드 — 편파적 브리핑

동일한 1주일 데이터를 상담사에게는 임상 톤으로 변환해서 보여준다. **상담사 뷰에서는 전문 용어를 사용한다** (상담사는 전문가이므로).

- 감정 패턴: "파국화 표현 3회, 수요일 이후 자기비하 증가 추세"
- 과제 이행률: "Adherence 60%. 산책 10분 0/5일, 감정 일기 4/5일"
- 미완료 맥락: "산책 미완료 시 '피곤해서 못 했다' 반복 — 회피 패턴 가능성"
- 세션 어젠다 제안: "긴급: 수요일 자해 충동 언급 / 중간: 수면 패턴 악화"

**반드시 포함: 디스클레이머**
브리핑 하단에 항상 다음 문구를 표시한다:
"⚠️ 본 브리핑은 AI가 생성한 참고자료이며, 임상적 판단을 대체하지 않습니다. 최종 판단은 상담사에게 있습니다."

**정량 데이터와 해석의 분리:**
- 과제 이행률, 표현 빈도 횟수 등 정량 데이터는 코드에서 직접 계산 (할루시네이션 없음)
- LLM은 패턴 해석과 어젠다 제안만 담당
- 정량 데이터 섹션과 AI 해석 섹션을 시각적으로 구분하여 표시

**편파적 브리핑의 기술 원리:**

동일 데이터 → LLM 시스템 프롬프트만 다르게 → 두 버전 생성.

내담자 프롬프트: "당신은 따뜻한 코치입니다. 공감적 톤으로, 임상 용어 없이..."
상담사 프롬프트: "당신은 임상 심리 어시스턴트입니다. 건조한 임상 톤으로, 전문 용어를 사용하여..."

→ 같은 데이터를 역할에 따라 다르게 해석하는 것은 LLM이 아니면 불가능하다.

### 4-5. 세션 연속성 타임라인

별도 통계/히스토리 탭 대신, 지난 3~4회 세션의 브리핑을 이어붙여 종단 패턴을 시각화한다.

- "3주 전: 직장 스트레스 → 2주 전: 가족 관계로 이동 → 이번 주: 자기비하 급증"
- 과제 이행률 추이도 함께 표시: "3주 전 80% → 2주 전 60% → 이번 주 40% — 하락 추세"
- 상담사가 흐름을 한눈에 파악, 내담자도 자신의 변화를 객관적으로 확인

→ 단순 14일 완료율 차트나 날짜별 히스토리보다 훨씬 가치 있는 뷰. 습관 앱의 통계가 아니라 "치료의 흐름"을 보여준다.

## 5. 화면 구조 요약

### 일반 모드 (3개 화면)
- 오늘 기록: 자유 텍스트 + 음성 입력
- AI 대화: 공감/코칭/응원/오늘의 질문 모드 선택
- 주간 리포트: 사고 패턴 미러링 + 감정 흐름

### 상담 모드 (5개 화면)
- 오늘 기록: 자유 텍스트 + 음성 입력 + 과제 체크리스트
- AI 대화: 동일
- 주간 리포트: 사고 패턴 + 과제 이행 패턴 통합
- 세션 준비: 내담자용 브리핑 / 상담사용 브리핑 (토글 전환)
- 타임라인: 세션 연속성 시각화

→ 탭이 적고 화면 전환이 적을수록 좋다. 하나의 화면에 관련 정보를 통합하는 것이 원칙.

## 6. 자연스러운 퍼널 (일반 → 상담 연결)

일반 모드를 쓰다가 특정 사고 습관이 3주 이상 반복되면, AI가 부드럽게 제안한다: "이 패턴이 계속되고 있네요. 더 깊이 탐구하고 싶다면, 전문가와 이야기해보는 것도 좋은 방법이에요."

**주의: 이것은 "진단"이 아니라 "제안"이다.** "당신에게 문제가 있다"가 아니라 "더 알고 싶으면 이런 방법도 있다"는 톤을 반드시 유지한다.

사용자가 상담을 시작하면 설정에서 상담 모드로 전환. 그동안 쌓인 기록 데이터가 그대로 상담사 브리핑에 활용된다. 처음부터 다시 설명할 필요 없이, 데이터가 이미 준비되어 있는 상태에서 상담이 시작된다.

## 7. MVP에서 빠지는 것 (이후 버전)

- SOAP 노트 자동 생성
- 위기 감지 / 긴급 개입 시스템
- 스마트워치 / 헬스킷 연동
- 동적 목표 협상 (수면 데이터 기반 과제 자동 하향)
- Voice-to-Voice 실시간 음성 대화 (MVP에서는 텍스트 기반 AI 대화, 음성은 입력만)
- B2C 유료 과금 모델
- 별도 통계 탭, 히스토리 탭 (주간 리포트와 타임라인이 대체)
- GTM/유통 전략 (이 프로토타입은 제품 가치 검증이 목적)

## 8. 데모 시나리오 (최종 시연용)

### 1단계: 일반 사용자 가치 (청중 모두가 공감)
- 발표자가 "저는 상담 안 받습니다"라며 일반 모드로 시작
- 월~금 감정 기록 입력 (텍스트 + 음성)
- AI 대화 시연 (공감 모드로 짧은 대화)
- "주간 리포트" 생성 → 사고 패턴 미러링 카드 표시
- 핵심 순간: "안 좋은 일이 생겼을 때 '최악의 결과'를 먼저 떠올리는 패턴이 이번 주 3번 있었어요. 정말 그렇게 될까요?"
- 오늘의 질문 카드 한 장 보여주기

### 2단계: 상담 연결 확장 (제품의 깊이)
- "근데 상담 받는 분은 이것도 됩니다"
- 상담 모드 전환 → 기록 화면에 과제 체크리스트 등장
- 주간 리포트에 과제 이행 패턴 추가된 모습
- 핵심 순간: "과제를 못 한 날에 자책하는 표현이 집중되어 있어요"

### 3단계: 편파적 브리핑 (하이라이트)
- 내담자용 세션 준비 브리핑 생성 (따뜻한 일상 언어)
- 토글 스위치로 상담사 뷰 전환 → 같은 데이터가 임상 전문 용어로 변환
- 정량 데이터(이행률, 표현 빈도)는 코드 계산, AI 해석은 별도 섹션
- 하단 디스클레이머 표시
- 이 토글 전환 순간이 데모의 하이라이트

### 4단계: 타임라인 (보너스)
- 3주치 세션 흐름 + 과제 이행률 추이를 시각화로 보여줌

## 9. 평가 기준 대응

### Is AI-Native? (5점 목표)

세 가지가 LLM 없이 불가능하다: (1) 비정형 텍스트에서 반복되는 사고 습관을 자동 감지하고 일상 언어로 설명, (2) 같은 데이터를 역할별로 다른 맥락의 요약으로 변환하는 편파적 브리핑, (3) 과제 미완료 메모와 감정 기록을 교차 분석해서 "왜 못 했는지"의 패턴을 읽어내는 것. 규칙 기반으로는 불가능한 기능들이다.

### So What? (5점 목표)

일반인: 매일 기록 + AI 대화 + 주간 사고 패턴 리포트 → 자기 사고 습관을 처음으로 인식. 오늘의 질문으로 매일 여는 습관, AI 대화로 체류 시간 확보.

상담 받는 사람: 과제 체크인으로 167시간의 데이터 축적 + 세션 전날 브리핑 → 50분을 더 효율적으로 사용. "왜 못 했는지"의 맥락까지 기록되어 세션에서 더 깊은 대화 가능.

상담사: 매 세션 전 브리핑으로 준비 시간 15분 → 3분. 과제 이행률(정량)과 패턴 해석(AI)이 분리되어 신뢰도 확보.

### What's New? (4~5점 목표)

감정 일기 앱은 많지만 반복되는 사고 습관을 자동 감지해서 일상 언어로 미러링해주는 앱은 없다. 습관 트래커는 많지만 "못 한 이유"와 "그때의 감정"을 교차 분석해주는 앱은 없다. 세션 밖 기록을 세션 전 양방향 브리핑으로 만드는 파이프라인도 시장에 없다.

### Why You?

팀원이 실제 상담을 받으면서 "매번 지난주 이야기를 처음부터 설명하는 게 시간 낭비", "과제를 했는지 안 했는지도 기억이 안 난다"고 느낀 경험에서 출발했다. 기획서에서 상상한 게 아니라, 상담실 안에서 직접 느낀 불편함을 해결하는 프로젝트다. MANIFEST.md에 이 경험을 구체적으로 쓸 것.

## 10. 예상 질문 방어

### "의료 행위 아닌가?"
이 앱은 진단 도구가 아니라 자기 인식 도구이다. 유저에게 임상 용어를 사용하지 않고, "당신은 OO 증상이 있다"가 아니라 "이런 표현이 반복되고 있어요"라고 패턴만 보여준다. 상담 권유도 "치료를 받으라"가 아니라 "더 알고 싶으면 전문가와 이야기해보는 것도 방법이에요"라는 제안이다.

### "AI 할루시네이션으로 잘못된 정보가 나오면?"
정량 데이터(과제 이행률, 표현 빈도)는 코드에서 직접 계산하므로 할루시네이션과 무관하다. LLM은 해석과 요약만 담당한다. 상담사 브리핑에는 항상 "AI 생성 참고자료" 디스클레이머를 표시하고, 최종 판단은 상담사에게 있는 구조다.
